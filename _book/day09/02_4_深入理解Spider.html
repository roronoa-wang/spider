课程整体概要

深入理解Spider类

Spiders定义了爬取网站的方式，包括爬取的动作（例如：是否跟进链接）以及如何从网页内容中提取结构化数据（提取item）。 换句话说，Spiders就是定义爬取的动作及分析网页的地方。

Scrapy框架提供了scrapy.Spider做为爬虫的基类，所有自定义的爬虫必须从这个类派生。scrapy.Spider类的主要属性和方法介绍如下。

（1）name属性

是定义爬虫名称的字符串。爬虫名称用于Scrapy定位和初始化一个爬虫，所以必须是唯一的。通常，我们使用待爬取网站的域名作为爬虫名称。例如，爬取mywebsite.com的爬虫通常会被命名为mywebsite。

（2）allowed_domains属性

包含了爬虫允许爬取的域名列表，是可选属性。

（3）start_urls属性

表示初始URL元组或列表。当没有指定特定的URL时，spider将从该列表中开始爬取。

（4）init()方法

初始化方法，负责初始化爬虫名称和start_urls列表。

（5）start_requests(self)方法

负责生成Requests对象，交给Scrapy下载并返回response。 该方法必须返回一个可迭代对象，该对象包含了spider用于爬取的第一个Request，默认是使用start_urls列表中的第一个URL。

（6）parse(self, response)方法

负责解析response，并返回Item或Requests（需指定回调函数）。返回的Item传给Item pipline持久化，而Requests则交由Scrapy下载，并由指定的回调函数处理（默认parse()）。然后持续循环，直到处理完所有的数据为止。

（7）log(self, message[, level, component])方法

负责发送日志信息。

scrapy.Spider的核心源码

"""
Base class for Scrapy spiders

See documentation in docs/topics/spiders.rst
"""

class Spider(object_ref):
    """Base class for scrapy spiders. All spiders must inherit from this
    class.
    """

    name = None
    custom_settings = None

    def __init__(self, name=None, **kwargs):
        '''
        初始方法: 对name,start_urls进行初始
        '''
        # 如果传入name不为None, 就是使用传入name给name属性赋值
        if name is not None:
            self.name = name
        # 如果name属性为None,就报ValueError的异常
        elif not getattr(self, 'name', None):
            raise ValueError("%s must have a name" % type(self).__name__)
        # 使用命名参数, 对象的属性进行更新
        self.__dict__.update(kwargs)

        # 如果没有start_urls属性, 就start_urls设置为空列表
        if not hasattr(self, 'start_urls'):
            self.start_urls = []


    @property
    # 属性方法, 用于创建一个本爬虫的日志对象
    def logger(self):
        logger = logging.getLogger(self.name)
        return logging.LoggerAdapter(logger, {'spider': self})

    # 用于输出日志信息
    def log(self, message, level=logging.DEBUG, **kw):
        """Log the given message at the given log level

        This helper wraps a log call to the logger within the spider, but you
        can use it directly (e.g. Spider.logger.info('msg')) or use any other
        Python logger too.
        """
        self.logger.log(level, message, **kw)

    # 负责根据start_urls中URL生成Request对象
    def start_requests(self):
        # 获取当前对象对应的类对象
        cls = self.__class__
        # 如果该类的make_requests_from_url方法被重写了, 就报一个警告
        if method_is_overridden(cls, Spider, 'make_requests_from_url'):
            warnings.warn(
                "Spider.make_requests_from_url method is deprecated; it "
                "won't be called in future Scrapy releases. Please "
                "override Spider.start_requests method instead (see %s.%s)." % (
                    cls.__module__, cls.__name__
                ),
            )
            # 遍历start_urls中的URL, 并通过make_requests_from_url方法生成Request对象, 交给scrapy引擎
            for url in self.start_urls:
                yield self.make_requests_from_url(url)
        else:
            # 如果make_requests_from_url没有被重写
            # 遍历start_urls, 生成Request对象, 把dont_fileter设置为true, 当Request中没有指明回调函数, 引擎默认交给Spider的parse方法进行处理
            for url in self.start_urls:
                yield Request(url, dont_filter=True)

    # 根据URL生成一个Request对象
    def make_requests_from_url(self, url):
        """ This method is deprecated. """
        return Request(url, dont_filter=True)

    # 用于解析数据和提取新的URL
    # 如果该方法没有被重写就报错
    def parse(self, response):
        raise NotImplementedError('{}.parse callback is not defined'.format(self.__class__.__name__))

面试题: 请说一下Scrapy入队的条件

  1. 一个全新的请求，能够入队
  2. dont_filter=True的请求能够入队
  3. start_urls中的url对应的请求，能够入队

总结(Spider类的运作流程):

  1. 引擎创建Spider对象,通过init方法,完成对name和start_urls的验证与初始化
  2. 引擎调用Spider对象的start_requests方法, start_requests方法根据start_urls中URL生成Request对象,交个引擎
  3. 引擎把Request对象交给调度器处理,当下载器空闲的时候, 引擎会从调度器中获取Request对象,经过下载中间件, 交给下载器下载, 下载器完成下载, 把响应数据封装为Response对象,经过下载器中间件交给引擎.
  4. 当Request中没有指明回调的方法的时候, 引擎会调用爬虫parse方法, 把响应对象交个爬虫进行处理. 如果在Request中指定了callback方法, 那么引擎就调用Request中指定的callback方法进行处理.
